Political Rebate - Application Edit Help



This screen provides the user with the ability to edit the details associated with an individual’s Rebate Application.

The following Actions (links) may be available from this screen. Remember that the actions that are available will depend on the user's privileges.

Reset (Button) The system clears any changes made to the screen returning it to its original content.
Cancel (Button) The system returns to the Political Rebate Application Screen without applying any changes.
Submit (Button) The system validates and applies any changes made.


All required fields must be entered, which includes at least one full Affiliate Identifier.

If the Return Date is populated, the Evaluation Code cannot be "Not Returned".

If the Return Date is greater than the system configured timeframe, usually 35 days, from the Mailed Date, the Evaluation Code is set to "Not Timely". The system configured timeframe must be set by a Administrator of the system.

The only allowable values for Affiliate Type under Dues Paid To is ‘L’ for Local, ‘R’ for Retiree Chapter, and ‘S’ for Retiree Sub Chapter.

If the Affiliate Finder functionality is not used to build the fields of the Affiliate Identifier and duplicate Affiliates are found for the data entered, the system will display the Duplicate Affiliate Identifier Notifier Screen as part of the flow to allow selection of the correct Affiliate.
