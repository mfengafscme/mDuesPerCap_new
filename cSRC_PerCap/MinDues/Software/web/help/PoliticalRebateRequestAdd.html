Political Rebate - Request Add Help



This screen provides the user with the ability to add a new Rebate Request that was submitted by an individual.

This screen allows the user to enter a date in the Rebate Year field and enter the Request date by clicking on the calendar icon (date picker) and then selecting a date.

Enter a Certified Mail Number, if available, in the associated field when applicable.

If checked, the Request Denied (checkbox) activates the Reason Denied pull down menu and Date Denied field with calendar icon (date picker).

The Keyed Date is generated by the system and cannot be edited.

The following Actions (links) may be available from this screen. Remember that the actions that are available will depend on the user's privileges.

Reset (Button) The system clears any changes made to the screen returning it to its original content.
Cancel (Button) The system returns to the screen from which this action originated.
Submit (Button) The system validates the data and submits the added Rebate Request.


The Rebate Year field is initialized as follows:
  • If from the Political Rebate Summary screen, the value is set to the same year as the Summary and cannot be edited.


  • If from the Political Rebate Summary By Year and the screen is part of the Add New Year flow, the value defaults to the previous calendar year but is editable and can be changed to a prior year.


    • All required fields must be entered, which includes at least one full Affiliate Identifier.

    The Keyed Date defaults to the current date and cannot be edited.

    If Request Denied is checked, Reason Denied and Date Denied become editable, the Reason Denied is required and the Date Denied is required and defaults to the current date.

    The lower section of the screen provides three 'Dues Paid To' fields and has the following sub categories: Type and State/National, Local/Sub Chapter and Council/Retiree Chapter. The Duration and Filed With fields also have pull down menus for making the appropriate selections. There is an additional field for Comments.

    The only allowable values for Affiliate Type under Dues Paid To is ‘L’ for Local, ‘R’ for Retiree Chapter, and ‘S’ for Retiree Sub Chapter.

    A Person/Member cannot request a Rebate for more than 3 Affiliates across all Requests for the Rebate Year.

    If the Affiliate Finder functionality is not used to build the fields of the Affiliate Identifier and duplicate Affiliates are found for the data entered, the system will display the Duplicate Affiliate Identifier Notifier Screen as part of the flow to allow selection of the the correct Affiliate.
