Political Rebate - Request Edit Help



This screen provides the user with the ability to edit a Rebate Request that was submitted by an individual.

This screen allows the user to enter the requested rebate year in the Rebate Year box and enter the Request date by clicking on the calendar icon (date picker) and then selecting a date.

Enter the Certified Mail Number (if available) in the designated field.

The Request Denied (checkbox), if checked, activates the Reason Denied pull down menu and Date Denied field with calendar icon (date picker).

The Request Re-submitted (checkbox), if checked, activates the Re-submission Denied pull down menu and Date Denied field with calendar icon (date picker).

The Keyed Date is generated by the system and cannot be edited.



The following Actions (links) may be available from this screen. Remember that the actions that are available will depend on the privileges given.

Submit (Button) The system validates the new data and submits the changes.
Reset (button) The system clears any changes made to the screen returning it to its original content.
Cancel (button) The system returns to the Political Rebate Request Screen without applying any changes.


All required fields must be entered, which includes at least one full Affiliate Identifier.

If Request Denied is checked, the Reason Denied and Date Denied fields become editable. In that case, the Reason Denied and the Date Denied inputs are required and defaults to the current date.

If Request Re-submitted is checked, Re-Submission Denied becomes editable. If Re-Submission Denied is populated, the Date Denied becomes editable and required and Date Denied will default to the current date.

The lower section of the screen provides three 'Dues Paid To' fields (the "Finder" links assists with retrieving the Affiliate Identifiers) and has the following sub categories: Type and State/National, Local/Sub Chapter and Council/Retiree Chapter. The Duration and Filed With fields have pull down menus for making the appropriate selections. There is an additional field for Comments.

The only allowable values for Affiliate Type under Dues Paid To is ‘L’ for Local, ‘R’ for Retiree Chapter, and ‘S’ for Retiree Sub Chapter.

If the Affiliate Finder functionality was not used to build the fields of the Affiliate Identifier and duplicate Affiliates are found for the data entered, the system will display the Duplicate Affiliate Identifier Notifier Screen as part of the flow for selecting the correct Affiliate.
